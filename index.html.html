SOM Design Decision

How to choose the grid size

The grid size controls how many geological facies the model can distinguish. Bigger isn't better — you need enough neurons to capture real geological variation, but not so many that the map becomes uninterpretable.

3×3
9 neurons
QE: ~1.8
Active facies: 3–4
Risk: too coarse
Merges distinct lithologies into single neurons. A sandstone and a shale at similar velocities look the same to the model.
✓ selected
4×4
16 neurons
QE FlatVel_A: 0.88
QE FlatFault_A: 0.44
Active facies: 5–9
Best balance. Enough resolution to separate soft sediments, compacted sediments, sedimentary rock, hard rock, and fault blocks. Geologically interpretable.
5×5
25 neurons
QE: ~0.31
Active facies: 10–18
Risk: overfragmented
Lower QE but splits what should be one geological unit into 3 neurons. Maps look noisy. Harder to inject meaningful context into the Vision LLM.
the three-criteria decision rule
1. QE elbow
Plot QE vs grid size. Pick the point where adding more neurons gives diminishing returns — the "elbow". For this dataset it's at 4×4.
2. Active neuron ratio
If >60% of neurons are active, the grid is well-utilized. If only 4 of 25 activate, you wasted 21 neurons. Target 40–70% active.
3. Geological meaning
Can you label each active neuron with a real lithology? Soft sed., hard rock, fault block? If yes, the grid is doing useful work.
Empirical starting point: neurons ≈ 5 × √(n_samples) → for 7000 vectors that's 415 neurons (20×20). We deliberately shrink to 16 for interpretability. This is a domain choice, not a statistics choice.

Architecture

3-layer pipeline

Each layer feeds context into the next. The SOM bridge is what prevents the Vision LLM from hallucinating faults in clean samples.

1
SOM — seismic facies classification
MiniSom 4×4 · 5000 iterations · QE=0.88 (FlatVel_A) / 0.44 (FlatFault_A)
Extracts 25 acoustic features per depth row for FlatVel_A (5 sources × mean, max, std, energy, zero-crossings), or 7 point-local features per pixel for FlatFault_A. Each position maps to its Best Matching Unit (BMU) in the 4×4 grid. The fault discontinuity score — mean absolute lateral difference across the lithology map — is injected into the Vision LLM prompt as prior context. Without this bridge, GPT-4o mini reported faults in 3/3 no-fault samples. With SOM context: 1/3 false positives.
$0.00
2
Vision LLM — geological interpretation
GPT-4o mini · detail:high · PNG→JPEG 1200px · SOM context injected
The 3-panel figure (seismic waveform, velocity map, SOM lithology) is compressed from PNG to JPEG at 1200px max width — cutting token usage 2.4× with no geological quality loss. The prompt includes the SOM fault score as structured prior: "CONTEXT FROM SOM: fault likely present (score=0.72, lateral index=34)". The model then confirms or contradicts the SOM finding from image evidence, producing a structured 5-point analysis of fault presence, velocity range, layer count, SOM colors, and seismic patterns.
~$0.003
3
RAG — scientific literature enrichment
FAISS local index · 60 FWI papers · all-MiniLM-L6-v2 · top-4 retrieved
Keywords extracted from the Vision LLM output (fault, layer, velocity) are used to query either the live arXiv API (v1) or a local FAISS index of 60 pre-downloaded FWI papers (v2). The 4 most relevant abstracts are injected into a second GPT-4o mini call (text only, no image) which writes a 200–250 word geological report with citations. The FAISS upgrade makes retrieval consistent across runs — the same 4 papers retrieved every time, not random arXiv results.
